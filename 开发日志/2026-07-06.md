## 图标问题完整记录 ⚠️ 重要

### 问题
iOS "添加到主屏幕" 显示默认图标（绿色背景 + 标题第一个字"动"），非自定义图标。

### 尝试方案（共 9 次）

| # | 方案 | 结果 |
|---|------|------|
| 1 | 相对路径 `<link rel="apple-touch-icon" href="./assets/icon-180.png">` | 失败 |
| 2 | RGBA → RGB（iOS 忽略透明图标） | 失败 |
| 3 | 换文件名 `icon-ios.png`、`icon-180.png` | 失败 |
| 4 | 绝对 URL | 失败 |
| 5 | 根目录 + assets 双路径 | 失败 |
| 6 | manifest.json 192+512 双图标 | 失败 |
| 7 | status-bar-style 改 black | 失败 |
| 8 | 删除 Service Worker | 失败 |
| 9 | 清除缓存 + 重启手机 | 失败 |

### 根因
**不是代码问题 — 是 GitHub Pages 的 Fastly CDN 与 iOS 图标请求不兼容。**
- 图标文件可访问（浏览器能打开）
- HTML 引用正确（验证过线上源码）
- **本地 Python 服务器成功过一次** — 证明代码和图标都没问题
- 同一套文件 **Netlify 部署立即成功** — 证明是托管平台差异

### 最终方案
**用 Netlify 代替 GitHub Pages。**
[app.netlify.com/drop](https://app.netlify.com/drop) 拖拽 `www/` 文件夹，2 秒部署，图标正常。

### 教训
1. iOS Safari 的 PWA 图标机制极其脆弱
2. GitHub Pages 不适合 PWA — CDN 层有兼容问题
3. 遇到顽固缓存直接换平台，别浪费时间硬调
4. 本地 Python `http.server` 是最干净的测试环境
